# 地下城跑酷骷髅主角与环境音设计

## 范围

本次改动仅包含两部分：

1. 将当前主角强化为“装备骷髅小杂兵”
2. 为游戏加入低沉地下城氛围环境音

不在本次范围内的内容：

- 不增加新的玩法按钮
- 不加入复杂战斗或攻击动作
- 不引入外部音频资源文件
- 不修改金币、障碍、计分的基础规则

## 目标

- 主角静止截图和运动中都能被明确识别为骷髅小兵
- 保留粗轮廓和小尺寸下的易读性
- 背景音乐更像地牢环境声，而不是旋律性 BGM
- 音频播放遵守浏览器交互限制，并且不打断核心跳跃操作

## 主角造型设计

- 主角继续保持约 `40x40` 的实体尺寸
- 识别重点放在“大骷髅头”，头骨占整体高度接近一半
- 头骨需具备以下强识别特征：
  - 明显的黑眼窝
  - 鼻孔形状
  - 简化但可见的牙槽
- 装备为辅助识别元素，不喧宾夺主：
  - 半头盔
  - 小圆盾
  - 短破披风
- 身体保持短小，避免做成普通灰色小兵
- 主色为骨白，描边为深灰黑，装备采用暗蓝灰或铁灰

## 主角动画与站位

- 继续使用“外层负责定位、内层负责动画”的双层结构
- 跑步动画只保留轻微起伏，不覆盖主角真实坐标
- 跳跃过程中头骨轮廓保持稳定，不因为动画失真
- 主角继续保持在画面左侧约 `18%` 的位置，不贴边、不压住 HUD

## 环境音设计

- 音频风格为地下城氛围环境音，不强调旋律
- 核心听感组成：
  - 低沉持续嗡鸣
  - 轻微风声或空气流动
  - 很弱的链条 / 石壁回响 / 金属余振
- 不加入明显鼓点或主旋律，避免与跳跃节奏竞争
- 开始页环境音更轻，开始游戏后稍微增强
- 游戏结束时快速淡出或明显压低

## 技术实现策略

- 新增独立音频模块，例如 `scripts/audio.js`
- 使用 `Web Audio API` 程序生成环境音，不依赖外部音频文件
- 在第一次用户交互后初始化 `AudioContext`
- 游戏状态只通过以下时机触发音频模块：
  - 开始游戏
  - 重新开始
  - 游戏结束
- 保留静音开关，避免用户被动持续播放

## 验收标准

- 主角在静态画面中一眼可认出是骷髅小兵
- 跑动和跳跃时主角不会重新退化成抽象灰块
- 开始游戏后环境音可以正常播放
- 结算时环境音有明显淡弱
- 不影响现有键盘/触控输入和游戏主循环
