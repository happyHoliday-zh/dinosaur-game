# 地下城跑酷首版视觉改版设计

## 范围

本次改版仅处理以下四项，不扩展到新玩法、音效或关卡系统：

1. 主角整体站位向右移动
2. 主角外观替换为粗轮廓小骷髅兵
3. 障碍物替换为更明确的怪物型障碍，并加入危险底座提示
4. 金币改为真正可见、可收集的实体

## 目标

- 让玩家一眼看出“哪个是主角、哪个是危险物、哪个是奖励物”
- 保持 Chrome 小恐龙式的单键跳跃直觉，不增加理解成本
- 在不更换技术路线的前提下，把当前原型提升到更像地下城题材的可玩版本

## 改动一：角色站位

- 主角的逻辑位置从当前偏左位置整体右移 `40-60px`
- 视觉目标是让主角落在画面左侧约 `18%` 区域，而不是贴近边框或 HUD
- 这一调整只修改角色的初始横向位置，不改变跳跃高度与碰撞尺寸基线

## 改动二：骷髅主角造型

- 主角继续使用约 `40x40` 的实体尺寸，不增大碰撞盒
- 造型改为“粗轮廓、易识别的小骷髅兵”
- 头部占比更高，眼窝与嘴槽清晰，身体短而稳
- 主色使用骨白，外描边使用深灰黑，辅色允许少量灰蓝或灰紫做披风/护甲点缀
- 跑步动画只做轻微起伏，不能影响主角实际位置

## 改动三：怪物障碍造型

- 障碍主力改成低矮怪物，不再依赖抽象陷阱轮廓
- 首版使用三类障碍：
  - 史莱姆：最低矮，亮绿或毒青
  - 骷髅犬：前倾冲刺轮廓，骨白头部配深色腿部
  - 持盾小怪：最方正，正面大盾牌，像“挡路墙”
- 每个障碍脚下统一加危险底座或暗红灰阴影圈，提升“这是碰撞物”的辨识度
- 障碍整体高度仍应满足当前单键跳跃规则，不引入高低层复杂判定

## 改动四：金币显示与收集反馈

- 金币从“分数阈值自动增加”改成场景内可见实体
- 金币视觉使用亮金色圆币、深色描边和中部竖纹
- 金币以 `3-5` 枚短串形式生成，放置在玩家起跳可碰到的高度
- 收集后金币立即消失，并附带短暂闪烁或缩放反馈
- 左上角 `COINS` 实时递增
- 首版金币仅承担累计奖励，不绑定护盾、宝箱或双倍分数

## 约束与兼容

- 保持当前原生 `HTML/CSS/JS` 实现方式
- 继续兼容 PC 键盘和手机点击操作
- 不修改当前开始、重开、分数、最高分等主流程
- 优先复用现有对象更新节奏，只新增必要的金币层与实体管理

## 实现建议

- 角色继续采用“外层负责定位、内层负责动画”的两层结构
- 障碍可继续复用现有障碍管理器，只替换怪物视觉类型与样式
- 金币建议新增独立实体层和轻量管理器，避免把奖励逻辑硬塞进障碍列表
- 碰撞策略保持矩形近似判定，首版优先稳定和可读性

## 验收标准

- 主角不会再贴边，看起来位于跑道内侧
- 主角视觉能被明确识别为小骷髅兵
- 障碍物即使静止截图也能看出“这是会撞死的敌人”
- 金币在画面中真实可见，可通过跳跃收集
- 收集金币时 HUD 数值即时变化
